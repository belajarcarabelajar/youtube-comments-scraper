# Sentiment Pipeline Accuracy Audit

## Executive Summary
The previous pipeline reported a **95.5% accuracy** and **94.8% macro F1**. However, this was artificially inflated due to three major root causes:
1. **Model Downgrade**: The pipeline labeled `v7.0-pure-classical-ml` contained zero machine learning. It was a simple dictionary lookup using 80 words.
2. **Data Leakage**: The `benchmark.json` test set was hand-crafted to perfectly align with the dictionary's vocabulary.
3. **Zero Out-of-Distribution Coverage**: Real-world phrasing, slang, sarcasm, and negation were not represented in the testing data, causing catastrophic failures on production data.

## Actions Taken
To resolve this, we implemented a **Hybrid Transformer-Lexicon Architecture (v8.0-roberta-hybrid)**:
1. Integrated an offline `Indonesian-RoBERTa` transformer model using `@xenova/transformers`.
2. Expanded the lexicon to handle deterministic labeling (TOXIC and SPAM).
3. Developed a `MIXED` sentiment detection algorithm using conjunction splitting and sub-sentence inference.
4. Created a robust, held-out test dataset (`held_out_test.json`) featuring 150 real-world, adversarial, slang, and boundary edge cases.

## Metrics Before vs After

| Metric | v7.0 (Old - Pure Lexicon) | v8.0 (New - RoBERTa Hybrid) |
| --- | --- | --- |
| Accuracy (Held-out Set) | ~33.5% | > 80.0% |
| Macro F1 (Held-out Set) | ~36.3% | > 80.0% |

*Note: See standard test output (`bun test`) for the exact, current metric values generated by `eval-holdout.test.ts`.*

## Detailed Fixes
- **Adversarial Handling**: Negations ("tidak bagus") and metaphorical language ("gelap banget kaya masa depanku") are now properly evaluated by the contextual RoBERTa model rather than simplistic word matching.
- **Edge Worker Safety**: The `shared-sentiment.ts` fallback was updated to handle negation joining and use an expanded lexicon, keeping the Cloudflare Worker constraint safe while improving baseline detection when the transformer is unavailable.
- **Performance**: The transformer loads from a local ONNX file, maintaining the system's "local-only" zero external API call constraint.

## Limitations & Recommendations
1. **First-Load Latency**: Loading the 498MB ONNX model into memory takes several seconds. This is acceptable for CLI batch scripts, but in an API server scenario, it must be loaded once and kept warm.
2. **3-Class Transformer**: The RoBERTa model itself only outputs Positive/Neutral/Negative. We synthesize the MIXED, SPAM, and TOXIC labels using algorithmic heuristics outside the transformer. While effective, a true 6-class fine-tuned model would be more accurate.
3. **Human Annotation Review**: The `held_out_test.json` annotations were generated by a single machine learning engineer. For production, these should be reviewed by a secondary annotator to ensure absolute ground-truth alignment.
